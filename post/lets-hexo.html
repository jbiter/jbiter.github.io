Enough talk. Let's Hexo!

概述

从去年就一直有搭建一个独立博客的想法，当时就开始接触了Hexo这款博客引擎，自己也在本地试玩了一下，但一直犯懒没有着手去做，这几天终于把博客搭建起来了。第一篇也不知道要写点儿什么，索性就先写一写我折腾着搭建博客的过程吧，希望能对有需要的你提供一些帮助。其实整个过程非常简单，自己摸索一下也完全可以搞定。

配置环境

Hexo是一款很优秀的静态博客引擎。它可以搭配很多类型的模板引擎，可以解析Markdown在内的多种文件类型，将它们输出为静态的HTML文件以供部署。你需要准备的环境有：

  • macOS/Linux/Windows
  • Node.js
  • Git

这里我使用的是MacOS，Git用于我们管理我们的博客项目和稍后配置自动发布。

安装Node.js

感谢Hexo，它是基于Node.js（以下称Node）的，这给我这个只会玩儿JavaScript的前端汪省了不少麻烦。如果你的电脑上已经安装了Node，这一步就可以跳过了。如果你还不知道Node是个什么鬼，那还愣着干啥，感紧去Google啊~

你可以在Node的官方下载页面根据你的操作系统选择对应版本下载安装即可。

当然如果你也跟我一样使用macOS，你也可以使用Homebrew来安装。

安装Git

如果你的电脑上已经安装了Git，那这一步也可以跳过。如果还没有，你可以先去下载安装，如果你还不熟悉Git的使用，你可以在这里找到Git的使用方法。

你一样可以使用Homebrew来安装Git

通过GitHub来管理你的Hexo项目

现在你的Hexo项目已经建立并且成功运行了，接下来是不是要开始愉快的写博客呢，先别着急。设想一下，你绞尽脑汁写了篇自认为非常满意的博客，为了吸引别人来看，还费了很大劲儿给自己的博客搞了“装修”，突然你的电脑炸了。那估计你想死的心都有了。所以需要把自己的代码托管在个靠谱的地方。GitHub就不错。
